## 製造固定費管理システム検討メモ

このリポジトリでは、製造固定費の予実管理システムに関する要件・設計メモを管理します。

> 注記: `製造固定費管理システム検討.pptx` は、この作業環境内では確認できませんでした。添付ファイルが利用可能になり次第、画面項目・レイアウト・業務フローを反映します。

### 想定用途

- 工場担当者が原価センタごとに支出予定・実績を登録する。
- 予実管理担当者が月末時点の支出予定を確認し、対予算変動や前月時点からの増減分析に利用する。
- Ariba発注データはCSV/Excel取込を基本とし、Ariba未対応サプライヤについてはシステム側で注文書PDFを発行する。

### 推奨アーキテクチャ

本番版は、ChromeとEdgeで動作するWebアプリとして構築し、会社標準に合う場合は以下の構成を推奨します。

- フロントエンド: React / TypeScript または ASP.NET Core Razor Pages
- バックエンド: .NET 8 / ASP.NET Core
- DB: SQL Server
- 認証: Microsoft Entra ID SSO
- 権限制御: 原価センタ単位の閲覧・編集権限、予実管理担当者権限、管理者権限
- 同時入力対策: SQL Serverのトランザクションと `rowversion` による楽観ロック
- 監査: 登録・更新・削除・ロック解除などの操作ログを監査ログテーブルに保存

100ユーザー、30原価センタ、約50,000レコード規模であれば、SQL Serverと一般的なWeb API構成で十分に運用可能です。

### 確定月ロックの制御案

- `accounting_period_locks` のような管理テーブルに、対象年月、ロック状態、実施者、実施日時、理由を保持します。
- 更新・削除APIは保存前に計上月のロック状態を必ず確認し、ロック済みであれば更新を拒否します。
- 画面側でもロック済みレコードの編集・削除ボタンを無効化しますが、改ざん対策として最終防衛線は必ずサーバー側に置きます。
- 管理者だけがロック解除または例外修正できるようにし、操作内容を監査ログに記録します。

### 今後確認したい要件

1. パワーポイント資料に含まれる入力画面・参照画面の項目、必須/任意、レイアウト、承認フローの有無。
2. Ariba CSV/Excelの実際の列名、文字コード、金額通貨、税抜/税込、明細単位。
3. 原価センタごとの閲覧・編集権限範囲と、兼務ユーザーの扱い。
4. 注文書PDFの正式レイアウト、採番ルール、社印・承認者表示の要否。
5. 実績データの連携元、月次締め日、ロック解除時の承認プロセス。
